Security & Configuration Architecture

┌─────────────────────────────────────────────────────────────────────────────────┐
│                    SECURITY & CONFIGURATION LAYERS                              │
│                                                                                 │
│  SECURITY (5 Layers)                                                            │
│  ┌─────────────────────────────────────────────────────────────────────────┐    │
│  │ 1. TLS/HTTPS                 OpenShift Ingress with Let's Encrypt       │    │
│  │    • Port 443 (encrypted)     • Certificate rotation (90 days)          │    │
│  │    • HTTP → HTTPS redirect    • HSTS headers enabled                    │    │
│  ├─────────────────────────────────────────────────────────────────────────┤    │
│  │ 2. Spring Security             HTTP Basic Auth (for API)                │    │
│  │    • PasswordEncoder: BCrypt   • Public endpoints: /search, /tags       │    │
│  │    • CORS: * origins (test)    • Protected: /, /actuator/*              │    │
│  │    • CSRF: Disabled (stateless API)                                     │    │
│  ├─────────────────────────────────────────────────────────────────────────┤    │
│  │ 3. Input Validation             Request validation via @Valid           │    │
│  │    • @Size, @Pattern, @PastOrPresent, @Positive                         │    │
│  │    • GlobalExceptionHandler catches validation errors                   │    │
│  │    • Returns HTTP 400 with detailed field errors                        │    │
│  ├─────────────────────────────────────────────────────────────────────────┤    │
│  │ 4. Data Sanitization           Elasticsearch DSL prevents injection     │    │
│  │    • Parameterized queries      • No string concatenation               │    │
│  │    • Query builder pattern      • Type-safe query construction          │    │
│  │    • Elasticsearch filters (immutable)                                  │    │
│  ├─────────────────────────────────────────────────────────────────────────┤    │
│  │ 5. Application Layer            GlobalExceptionHandler                  │    │
│  │    • Exception handling         • No stack traces in production         │    │
│  │    • Error response standardization  • Logging all errors               │    │
│  │    • Rate limiting ready (Spring Cloud Gateway)                         │    │
│  └─────────────────────────────────────────────────────────────────────────┘    │
│                                                                                 │
│  CONFIGURATION MANAGEMENT                                                       │
│  ┌─────────────────────────────────────────────────────────────────────────┐    │
│  │ DEVELOPMENT (application-dev.yml)                                       │    │
│  │ • spring.profiles.active: dev                                           │    │
│  │ • Logging: DEBUG                                                        │    │
│  │ • Hibernate: update (auto-create schema)                                │    │
│  │ • PostgreSQL: localhost:5432                                            │    │
│  │ • Elasticsearch: http://localhost:9200                                  │    │
│  │ • Redis: localhost:6379                                                 │    │
│  ├─────────────────────────────────────────────────────────────────────────┤    │
│  │ STAGING (application-staging.yml - Not shown, recommended)              │    │
│  │ • Logging: INFO                                                         │    │
│  │ • Hibernate: validate                                                   │    │
│  │ • Metrics enabled                                                       │    │
│  ├─────────────────────────────────────────────────────────────────────────┤    │
│  │ PRODUCTION (application-prod.yml)                                       │    │
│  │ • spring.profiles.active: prod                                          │    │
│  │ • Logging: WARN (critical only)                                         │    │
│  │ • Hibernate: validate                                                   │    │
│  │ • All credentials via environment variables:                            │    │
│  │   - DB_URL, DB_USER, DB_PASSWORD                                        │    │
│  │   - ELASTICSEARCH_URL                                                   │    │
│  │   - REDIS_HOST, REDIS_PORT, REDIS_PASSWORD                              │    │
│  │ • Connection pools at max capacity                                      │    │
│  │ • Compression enabled                                                   │    │
│  │ • Metrics exported to Prometheus                                        │    │
│  └─────────────────────────────────────────────────────────────────────────┘    │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
